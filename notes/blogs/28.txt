Web 会议系统

有时候地理位置不在同一地方的几个人想开会，讨论个设计，开个讲座啥的，有个 Web 会议系统会很方便，客户端有浏览器就行，装个 JRE 或者 Flash 插件。Web conferencing 系统的几个基本需求：

    客户端只需要 Web 浏览器，一般都会用 Flash 或者 JRE 插件，Google 搞的 WebRTC 基于 HTML 5，很有意思。
    会议室的权限控制：谁可以参加，谁可以得到主持权力，谁可以邀请别人参加，等等。
    共享白板用来画示意图
    共享屏幕用来展示一些文档什么的
    共享桌面控制，允许别人暂时操作下自己的机器
    音频、视频通讯，声音是不用说了，视频可以用来看看说话的人是谁（我经常分辨不出电话会议里说话的人是谁。。。）
    会议过程的录制，供事后查阅
    跟日历系统集成，或者有自己的日历系统，查看有哪些会议即将发生，过去开过什么会议
    会议的进行跟某一个参与者是否在线没有耦合，比如主持人掉线了，会议不能自动结束，会议录制不能中断
    为了避免泄漏商业机密，Web 会议系统不能是第三方公司管理的

商业 Web 会议系统很多，我用过 Adobe Connect，听说过 Lync 和 WebEx。满足部分需求的产品也颇有一些，比如 NetMeeting(可惜 Ekiga 不支持白板、程序共享、桌面共享）、一些 IM 工具。开源的正经 Web 会议系统却是很少，很可能是因为涉及到音频、视频、流媒体、客户端 JS、Flash 等多方面技术，不那么容易实现。从网上打听到的开源 Web 会议系统有这些：

    OpenMeetings：原来代码在 google code 上，现在转移到 Apache 组织下孵化了，这是我唯一试用过的，运行很方便，开箱即用，外部依赖极少，功能比较全面，基本可用，小 bug 比较多。客户端用的 Flash 做界面，Java WebStart 技术做屏幕共享，服务端使用 Java 技术，尤其是 Red5 流媒体服务器。
    BigBlueButton：开发状态不错，但我看到安装文档就退却了，安装还真罗嗦，依赖暴多，看文档似乎是混杂了 Ruby/Java/Flash。看 demo 的界面貌似功能不如 OpenMeetings 丰富。
    WebHuddle:  这厮号称是开源的，其在 sourceforge.net 上的源码看起来很久没更新了，下载里有 0.4.9 Windows 版，2007 年上传的。从这个台湾兄弟的教学片看(http://163.13.177.120/blog/index.php?load=read&id=11)，这套系统服务端用了 jboss，客户端是用 Java Applet。


总结下：如果企业有这方面的需求，还是找个商业系统吧，实在没钱又不想盗版，可以试试 OpenMeetings。
