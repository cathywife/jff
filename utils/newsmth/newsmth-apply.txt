1. 精华区整理: 目标是尽量将主题放到一级目录, 由于
BBS上的回答大多简短, 所以一级目录相当于FAQ索引.
分类如下(具体命名待定):

			  |-- tips/

shell, sed, awk相关/(Perl, Python都有专版)

              |-- Debian
              |-- Fedora Core
              |-- ...
版务相关/
INDEX
ChangeLog
README

关于各级目录下几个特殊文件的说明:
每一级目录下包含INDEXnn文件, 这个文件包含了此目录下所有文件的
索引, 内容是帖子标题以及摘要, 这个文件每到一定大小就增加一个;
每一级目录下包含README文件, 包含此目录主题详细说明以及其它未尽
事宜.

根目录下包含INDEX文件, 包含所有子目录下的索引, 这样做的目录是
便于类似rss的获取精华区更新(新水现在有这个功能了吗?).
根目录下包含ChangeLog文件, 包含精华区修改记录.
根目录下包含README文件, 说明第一级目录的分类标准, 列举关键词.


2. 版面管理
本着尽量不封人的原则，尽量引导版面讨论深入，规范提问与回答(
只是建议提问标题清晰，回答者给出资源索引，不要多次跑题太远)。

由于我现在上班，周末休息不便上网(请网友谅解)，而且大家灌水的欲望
总是有的，适当放松一下还是可以的，所以计划周一至周五较严，周六周日
较松，此时谈论新闻，发表笑话，臆测，传言，牢骚，都可以，只要跟Linux
扯得上边，当然不要太过分了，显而易见的灌水是不允许的.

周一至周五，上述不太切题的话题我会设置不可回复，并给出解释，
新闻，笑话等不可回复，并转贴、置底。如果对此有异议请发信给
我，我开一个投票箱，如果五人以上不同意，我撤销不可回复。
需要删文时转到精华区临时目录，同时发信通知，如有异议，请发信给我，
我开启投票，如五人以上不同意，我转回版面(不知道新水能不能把精华区
文章转到版面，不行的话我手工贴一份).
// 但愿投票箱够用...

尝试写个脚本统计版面, 统计每人发文数, 每人发文总分数(正数, 五分制,
累加, 标准暂时我评估, 大家可以找个时间讨论), 每人发文回文总分数, 
每人发文总水分数(负数, 五分制, 累加), 每人发文回文总水分数(负数, 
五分制, 累加), 这些数据可以用来评估版面质量以及我的工作质量, 每月
评选最佳新人奖(分数最高新人), 最佳老新人奖(分数最高的不是那么新的新人),
最佳新老鸟奖(分数最高的不是那么老的老鸟), 最佳老鸟奖(分数最高老鸟),
等等等等. 
分数分成两种是为了不打击网友发文积极性, 你可以看到自己的贡献, 也能
看到自己的水文量, 一码归一码. 这些分数也供我参考删文以及不可回复的
力度.

评分工作肯定会增加工作量, 所以后面谈网友参与时请大家合作帮忙, bow!

鼓励提问者在问题解决后写总结, 给分奖励, 总结格式如下:
标题: 什么问题
摘要: 摘要
详细说明: xxxxx
特别对于长的主题, 总结一下很有必要, 这些总结放入精华区后便于查阅(INDEXnn
文件会记录标题和摘要), 也算受人恩惠泽及后人吧, 希望大家支持.

鼓励提问者描述清楚问题, 说的越清楚, 分越高.

所有统计分数我会公布出来, 有兴趣的可以查对, 不满的请发信给我:-)

有关*版务*问题的来信我必回复, 但不保证能给各位满意的答复, 希望大家谅解!

提倡专题讨论, 专题来源可以各位网友建议, 我也尽力总结版面上最近常见
问题, 做一个专题深入讨论, 每次开启投票箱选择下一阶段做什么专题, 各位
报名发稿, 讨论都可以, 具体安排待定.

鼓励自己查资源比如google, 手册文档甚至源码解决问题, 严重加分.

如果计划理想达到, 有些成果出来, 可以考虑版聚, 坚决不做饭帮:-)
